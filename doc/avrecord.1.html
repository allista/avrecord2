avrecord


Updated: September 17, 2011

NAME

avrecord - program for capturing video and audio upon motion or noise detection

 

SYNOPSIS

 

Record mode:

avrecord [-c config_file] [-l log_file] [-ndh?]  

Configuration initialization mode:

avrecord -i[device_name] -t config_file_template -o new_config_output

 

DESCRIPTION

avrecord is an audio-video recorder with motion and noise detection. It constantly monitors a video stream from a v4l device looking for difference between consequent frames. It may also monitor sound level in a stream captured by a soundcard. While waiting for motion or noise it buffers data. When something is detected buffered data is written out so the precise moment of the event's beginning is not clipped.

 

CONFIGURATION

Before you start using avrecord for the first time you should initialize it's configuration using the template file provided (see FILES section). Configuration is initialized by quering your video device capabilities and controls and adding them to the main configuration structure. Initialized configuration is saved then to a new file.

 

OPTIONS

 

Record mode:

-c config_file
Specify alternative configuration file.
-l log_file
Specify alternative log file.
-n
Log to stdout.
-d
Start in daemon mode.
-h, -?
Print command help.

 

Configuration initialization mode:

-i[device_name]
Initialize configuration. If you specify a device name, please be carefull: there must be no space between the option and the name (e.g. -i/dev/video0) If no device name is provided the one specified in a template file is used.
-t config_file_template
Use specified configuration file as a template for a new configuration.
-o new_config_output
Write initialized configuration to the file specified.

 

FILES

/etc/avrecord/avrecord.cfg.template provides configuration skeleton for the first initializatioin of a configuration. All static configuration options are documented there.

./avrecord.cfg, ~/.avrecord/avrecord.cfg, /etc/avrecord/avrecord.cfg configuration files in the order they're loked for by avrecord if the -c option is not used.

./avrecord.log default log file. Usualy log file path is defined in the configuration or by the -l option though.

/etc/init.d/avrecd daemon startup script. It may not work in your particular linux distribution due to startup system differences. It's there mainly for tailoring by distribution maintainers and system administrators.
